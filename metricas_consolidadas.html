Contexto de la prueba

Descripción de los desarrollos evaluados y las condiciones de la prueba

Se evaluó el ciclo completo del pipeline IA sobre un conjunto de tareas de desarrollo que atraviesan las tres áreas del proyecto. Las pruebas se realizaron sobre los siguientes tickets:

  • API para alta de cliente y operación — DCM-96 (Nuevas APIs).
  • Modificación de APIs de Loan — NPA-283 (Modificación de APIs existentes).
  • Incorporar parámetro a la opción de informe Deudores BCRA — DM-1443 (Generación de archivos informe BCRA).

El flujo comienza con Análisis Funcional generando el DCR a partir de material de relevamiento, continúa con Software Factory desarrollando el código N-Tier desde el DCR, y finaliza con Quality Assurance generando casos de prueba y automatización a partir del mismo documento.

Cada área utilizó Claude como modelo de IA, integrado a través de agentes especializados y skills configurados para las tareas específicas de cada rol. Las métricas miden el tiempo empleado antes y después de la incorporación de la IA al proceso.

Conclusiones

Resultados consolidados de la prueba

  • La incorporación de Claude como IA asistente logró una reducción promedio del 90% en los tiempos de las tareas principales de las tres áreas.
  • El pipeline completo AF → SWF → QA se ejecutó de extremo a extremo, validando la viabilidad del flujo integrado con IA.
  • La cobertura de requerimientos, tests y escenarios alcanzó el 100% en las tres áreas, superando los niveles históricos del proceso manual.
  • Los artefactos generados (DCR, código VB.NET, scripts SQL, casos de prueba Excel y automatización Java) fueron consistentes y listos para revisión.
  • Los equipos pueden concentrarse en las decisiones complejas y de alto valor, delegando las tareas repetitivas y estructuradas a la IA.

Nota: Este informe será actualizado por cada área a medida que se completen las pruebas con datos reales. Los gráficos y las métricas se ajustarán automáticamente a los nuevos valores.
