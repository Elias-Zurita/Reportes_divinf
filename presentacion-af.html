Agentes especializados

Revisor de Código y Reglas de Negocio

Cada agente delega a skills especializados según tipo de requerimiento o módulo involucrado.

🔍 RevisorCodigo Relevamiento técnico
Evalúa el tipo de requerimiento y activa skills según corresponda: API, Pantalla, Batch, Políticas, Artículo.
Skills: entidades presentacion procesos-batch api loan-components loan-policies loan-profiles loan-articulo-formas-pago-solicitud
📋 ReglasNegocios Reglas de negocio e interfaz
Evalúa los módulos involucrados y activa skills según corresponda: Cobranzas, Facturación, Producto, etc.
Skills: loan-collections loan-debit loan-billing loan-settlement loan-transferred-portfolio loan-commercial-policies loan-articulo loan-commercial-plan loan-profiles loan-levels

Documento Funcional

Input: DCR → Output: Documento HTML

El comando /documento-funcional toma un DCR existente y genera el documento funcional presentable al cliente.

📄 DCR.md Documento de Cambio Requerido
INPUT
⚙️ documento-funcional Skill: DCR → JSON
PROCESO
📋 documento-funcional.html
📦 JSON (datos)
OUTPUT

Comando: /documento-funcional @DCR-{ticket}.md

Mapea DCR a JSON según schema, aplica reglas de redacción para cliente y produce HTML autocontenido exportable a PDF.

Outputs

Cómo usar

Recolección y organización del material

Antes de ejecutar /analisis, reunir todo el material de relevamiento en una carpeta.

📋

Material a reunir

Minutas de reuniones, documentos de propuestas, requerimiento formal, apuntes y notas del relevamiento.

📁

Organizar en carpeta

Crear carpeta con nombre del ticket (ej: NPA-283). Incluir minuta-*.txt, message.msg, requerimiento.md, apuntes-*.txt.

▶️

Ejecutar /analisis

/analisis docs/requerimientos/NPA-283 — Valida, genera DCR y opcionalmente /documento-funcional.
