Áreas del proyecto

Seleccioná un área para acceder a sus reportes de avance. El flujo comienza con el DCR generado por AF.

01 Primer paso
Análisis Funcional

AF — Análisis Funcional

El área de Análisis Funcional es responsable de generar el DCR (Documento de Cambio de Requerimiento) que da inicio al flujo del proyecto IA. Define los requisitos que guían el desarrollo y las pruebas.

Software Factory · Desarrollo

SWF — Software Factory

El área de Software Factory toma el DCR generado por AF y lleva a cabo el desarrollo de las funcionalidades requeridas. Sus entregas son la base sobre la que QA ejecuta las pruebas.

Quality Assurance

QA — Quality Assurance

El área de QA toma el DCR de AF y el desarrollo de SWF para generar casos de prueba y tests automatizados mediante agentes de IA, cerrando el ciclo del proyecto.
