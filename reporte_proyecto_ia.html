El Desafío

El problema que debemos resolver

El proceso actual de generación de artefactos QA presenta múltiples fricciones que impactan la velocidad y calidad de entrega.

⏳
Alto tiempo manual
Crear casos de prueba a partir de un DCR requiere horas de análisis, redacción y estructuración manual por parte del equipo de QA.
⚠️
Inconsistencia en la cobertura
Cada analista interpreta el documento de forma diferente, generando gaps en la cobertura de pruebas y escenarios no contemplados.
🔄
Trabajo repetitivo y propenso a errores
La transformación de requerimientos a código de automatización es un proceso repetitivo que consume tiempo valioso del equipo técnico.
🐌
Ciclos de entrega más lentos
El bottleneck en la preparación de QA retrasa el ciclo de validación, impactando los tiempos de release del producto Loan Software.

Input del Sistema

El DCR: punto de partida de todo

El Documento de Cambio/Requerimiento (DCR) es el insumo principal que el sistema consume para generar todos los artefactos de calidad.

📄 dcr.md — Documento de Cambio/Requerimiento Análisis Funcional
📌 Descripción del cambio
🎯 Requerimientos funcionales
✅ Criterios de aceptación
⚠️ Consideraciones especiales
👥
Creado por el equipo de Análisis Funcional
El DCR es producido por el área de AF que describe en lenguaje de negocio todos los cambios, reglas y validaciones del sistema.
📝
Formato Markdown estructurado
El archivo dcr.md utiliza formato Markdown, facilitando su procesamiento automatizado y la extracción de información semántica.
🔗
Contiene toda la información necesaria
Incluye descripción del cambio, requerimientos funcionales, flujos de usuario, criterios de aceptación y restricciones del negocio.
🚀
Un solo archivo, múltiples artefactos
A partir de este único documento, el sistema genera automáticamente tanto los casos de prueba como el código de automatización.
Agentes IA

Los agentes que hacen el trabajo

Cada agente es una unidad inteligente especializada en una tarea específica del proceso de generación de QA.

#01
🔍
Agente Analizador de Requerimientos
Lee e interpreta el DCR completo. Identifica actores del sistema, flujos de negocio, reglas de validación, módulos impactados (Operaciones, Cobranza, Facturación) y pre/post condiciones de cada requerimiento.
Skill Interpretación NLP Análisis
#02
📝
Agente Generador de Casos de Prueba
Transforma los requerimientos analizados en casos de prueba estructurados: escenarios positivos y negativos, datos de entrada/salida, condiciones de borde y criterios de aceptación verificables.
Skill Escenarios Skill Generación
#03
⚡
Agente de Automatización
Convierte los casos de prueba en clases Java funcionales, utilizando el framework de automatización existente (Playwright/Selenium, TestNG, Allure). Genera código listo para ejecutar en el proyecto loan-auto-tests.
Skill Código Java Framework Integration
Skills

🧠

NLP
🎯

Orquestador

El cerebro del sistema

El orquestador es el componente central que coordina y controla todo el flujo de trabajo, garantizando que cada agente opere en el momento y contexto correcto.

🎯 Orquestador Motor de coordinación
📥
Recepción y parseo del DCR
Ingesta el archivo dcr.md, valida su estructura y prepara el contexto inicial para los agentes.
🔀
Coordinación del flujo entre agentes
Determina el orden de ejecución, pasa el contexto entre agentes y gestiona las dependencias de datos.
🛡️
Manejo de errores y reintentos
Detecta fallas en agentes, aplica políticas de reintento y garantiza la integridad del proceso completo.
📤
Consolidación y entrega de outputs
Reúne los resultados de todos los agentes y genera los artefactos finales: el Excel y las clases Java.
